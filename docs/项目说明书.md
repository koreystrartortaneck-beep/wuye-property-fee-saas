## 一、项目概述

一套面向**多家物业公司**的智慧物业 SaaS 系统。物业公司用它管理旗下多个小区的物业费收缴与社区服务；业主用微信小程序缴费、报修、看公告、约服务。

**核心定位**：不是"单小区工具"，而是**多租户 SaaS 平台**——一套系统服务多家物业公司，每家公司管理多个小区，公司之间数据物理隔离。

**三类角色 / 三个终端**：

| 角色 | 终端 | 用途 |
|---|---|---|
| **业主** | 微信小程序 | 缴物业费、报事报修、访客邀请、看公告、约生活服务 |
| **物业管理员** | Web 管理后台 | 配置收费规则、导房产、出账、审核、发公告、接单 |
| **平台超管** | Web 管理后台 | 开通/管理物业公司（租户），跨租户视角 |

## 三、功能清单

### 业主端（微信小程序 · 22 页）

底部四个 Tab，遵循「看 / 办 / 账单 / 我的」四分：

**🏠 首页（看近况）**
- 缴费卡：本期待缴/已缴一目了然，一键缴纳
- 社区动态：物业公告 + 工作照片混排信息流（首页显示最近 3 条摘要）
- 多房屋切换（一个业主可绑多套房、跨小区）

**🔧 服务（办事）**
- 报事报修（拍照上报，最多 3 图）
- 投诉建议
- 生活服务预约（保洁/清洗等上门，带价格）
- 访客邀请（生成 6 位通行码）
- 联系管家（一键拨号）

**📄 账单**
- 按账期分组 + 费用科目筛选
- 账单详情（完整计费依据：单价×面积、抄表读数、公摊方式）
- 合并缴纳、电子收据

**👤 我的（账户/记录）**
- 我的房屋（绑定/切换）
- 我的工单、我的预约、缴费记录

**其他**：绑定房屋（手机号自动匹配 + 自助申请审核）、社区动态档案页、工作照片墙、卡券中心（已开发，当前入口下线）

### 物业管理后台（Vue3 · 17 页）

- **收缴看板**：应收/实收/收缴率，按小区分行
- **小区管理 / 房产管理**：CSV 批量导入房产
- **收费规则**：五类计费（单价×面积 / 固定 / 抄表 / 公摊 / 自定义公式）
- **抄表录入 / 公摊录入**：含"未录清单"
- **出账与账单**：手动/自动出账（幂等）、账单查询作废
- **绑定审核**：业主申请通过/驳回
- **报事报修**：受理派单/办结/评分查看
- **社区公告 / 工作照片墙**：发布、图片上传
- **生活服务**：服务菜单管理 + 接单
- **访客通行**：输码核销
- **优惠券**：发券 + 核销
- **通知记录**、**租户管理**（超管）

### 平台端（超管）

- 开通物业公司（租户），自动创建管理员账号
- 顶栏切换租户视角，跨公司管理

## 四、核心机制（技术亮点）

### 1. 多租户数据隔离（安全底座）

所有业务表带 `tenantId`，通过 Prisma 数据库中间件**自动注入租户过滤**——业务代码写不出跨租户查询。有专项测试保证"A 物业绝对看不到 B 物业的任何数据"。

### 2. 计费引擎（灵活可配）

支持五类收费规则，物业在后台自行配置：

| 规则 | 说明 |
|---|---|
| 单价×面积 | 物业费主力，如 15 元/㎡/月 |
| 固定金额 | 车位费、垃圾清运费 |
| 按表计量 | 水电燃气代收，抄表读数×单价 |
| 公摊分摊 | 公共能耗按面积/按户分摊（最大余数法，总额守恒） |
| 自定义公式 | 表达式安全求值，白名单变量 |

金额全程用**整数分**计算，杜绝浮点误差。

### 3. 自动出账 + 催缴

- 每日 02:00 定时扫描，按规则周期自动出账
- 幂等设计：重复触发只补缺，绝不重复出账
- 每日 09:00 扫描：到期前 3 天提醒、逾期提醒（去重）

### 4. 支付层可插拔

支付做成接口，当前为 **Mock 支付**（演示用，点确认即成功、不真扣钱）。接真实微信支付只需实现一个 Provider + 填商户号，业务逻辑零改动。

### 5. 微信能力双模式

微信登录、手机号、订阅消息均为 `mock/real` 双实现，环境变量切换，本地/体验版全流程可测。

## 八、产品设计原则（贯穿全程）

1. **信息架构：看 / 办 / 记录 三分离**——首页看近况、服务办事情、我的存记录，各归其位，不堆砌。
2. **首页是仪表盘**——只放摘要，全量进列表页，无论内容多少首页高度稳定。
3. **数据错位比加载慢危险**——切换房屋先清旧数据，宁可闪一下空态也不张冠李戴。
4. **诚实落地**——不做依赖外部资源的空壳功能（如优惠券改为物业自发券）。
5. **视觉统一**——墨紫做骨、象牙做纸、藕灰做肉、香槟金点睛。
