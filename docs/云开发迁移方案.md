# 迁移到微信云托管（保留全部后端代码）方案

> 目标：小程序改用 `wx.cloud.callContainer()` 走微信内部通道——**免备案、免域名、可发正式版**，且**不依赖自有服务器**。
> 决策依据：云开发的 MySQL 只能用数据模型 SDK（Prisma 连不了，要重写两周）；**云托管同样免备案，但给标准 MySQL 连接串，Prisma/NestJS/118 测试全保留（~1-2 天）**。
> 前提：企业主体已就绪，AppID `wx9e8f35712e4bc1d3`，云环境 `wuye-api-d2gql1e3g718d01c9`

---

## 目标架构

```
业主小程序 ──wx.cloud.callContainer()──→ ┐
              (免备案内部通道)             ├─→ 云托管服务（我们的 NestJS 容器）──→ 云托管内置 MySQL
管理后台Web ──公网HTTPS域名(云托管提供)──→ ┘
```

- **后端容器原样搬**（deploy/Dockerfile.api 直接用）
- 数据库用**云托管内置 MySQL**（标准连接串，Prisma 照用）
- 彻底不依赖自有服务器

## 几乎全保留，只改一处

| 保留不动 | 需要改 |
|---|---|
| NestJS 全部代码（15 模块 / 22 表 / 118 测试） | 小程序 `utils/request.js`：`wx.request` → `wx.cloud.callContainer`（封装层吸收，页面不动） |
| Prisma / 计费引擎 / 多租户隔离 | 管理后台 `api.ts`：API 地址指向云托管公网域名 |
| Dockerfile / compose | 环境变量 DATABASE_URL 指向云托管内置 MySQL |

## 分阶段实施

### C0 · 开通云托管 + 建服务 + 建内置 MySQL【你操作，我指引】
- 微信云托管控制台 → 新建服务（如 `wuye-api`）
- 开通云托管**内置 MySQL 数据库** → 拿到 `MYSQL_ADDRESS / MYSQL_USERNAME / MYSQL_PASSWORD`
- 记下服务的**公网访问域名**（云托管自动分配，含 HTTPS）

### C1 · 部署容器【我操作】
- 云托管支持从 **GitHub 仓库**（我们已有）或本地代码构建
- 用 `deploy/Dockerfile.api`；容器监听端口设 3000
- 配环境变量：`DATABASE_URL=mysql://用户:密码@MYSQL_ADDRESS/property_fee`、`JWT_SECRET`、`WX_MODE`、`PAY_MODE` 等

### C2 · 数据库迁移【我操作】
- 容器启动自动跑 `prisma migrate deploy`（已内置在 Dockerfile CMD）
- 跑 seed 演示数据

### C3 · 小程序改造【我操作】
- `utils/request.js` 封装成 `wx.cloud.callContainer({config:{env}, path, method, header, data})`
- NestJS 路由（/api/v1/...）原样работает；页面调用签名不变
- 上传体验版，全流程回归

### C4 · 管理后台【我操作】
- `api.ts` 的 API base 指向云托管公网 HTTPS 域名
- 静态文件托管：暂留自有服务器 / 或迁云开发静态托管（后定）

### C5 · 发正式版【你+我】
- 全流程回归 → 提交审核（物业类目按需补资质）→ 发布
