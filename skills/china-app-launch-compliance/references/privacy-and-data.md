# 个人信息与数据

## 数据盘点

建立字段级清单：字段、来源、目的、必要性、处理依据、保存位置、期限、共享对象、删除方式和安全措施。至少检查手机号、OpenID/UnionID、姓名、身份证明、房屋地址、业主/住户关系、账单、欠费、支付订单、发票信息、设备标识、日志和客服记录。

## 重点判断

- 区分个人信息、敏感个人信息、匿名化数据和业务记录；房屋地址、身份关系、金融账户等组合可能显著提高风险。
- 处理敏感个人信息时核验特定目的、充分必要性、严格保护措施、单独同意和影响告知等适用要求。
- 核验告知文本与实际代码、接口、日志、埋点和第三方 SDK 行为一致。
- 第三方 SDK 必须列明名称、主体、目的、数据类型、隐私政策链接和触发时机；不得只复制 SDK 厂商模板。
- 未成年人、跨境提供、委托处理、共同处理、对外提供、公开披露和自动化决策分别做条件判断。
- 高风险处理活动核验个人信息保护影响评估及记录保存要求。

## 用户权利与安全

检查查阅、复制、更正、补充、删除、撤回同意、注销、解绑房屋、投诉和响应流程。明确法定留存数据与注销后继续保存数据的范围、理由和期限。检查最小权限、分级授权、传输/存储保护、日志脱敏、备份删除和安全事件预案。
