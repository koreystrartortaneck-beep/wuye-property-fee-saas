# AI解题小程序 —— 服务号H5上线完整指南

> 调研日期：2026-07-03  
> 产品形态：AI解题工具 + 会员订阅收费（月费/年费）  
> 技术方案：服务号内嵌H5网页 + 调用第三方大模型API  
> 核心问题：走服务号H5到底要准备什么？和小程序有什么不同？

---

## 一、H5 vs 小程序：关键差异一览

| | 微信小程序 | 服务号H5网页 |
|---|:---:|:---:|
| **微信代码审核** | ✅ 严格审核，会检查类目 | ❌ 不审核你的web代码 |
| **深度合成类目** | 必须补充，否则驳回 | 不存在"类目"概念 |
| **算法备案材料** | 必须上传给微信 | 微信不查 |
| **支付方式** | 微信虚拟支付；iOS走Apple订阅(12%) | 微信JSAPI支付，**无Apple税**（费率0.6%） |
| **用户登录** | `wx.login` 静默获取openid | 网页授权 OAuth，需用户点同意 |
| **入口体验** | 下拉菜单/桌面快捷/模板消息 | 服务号菜单/推文/聊天分享 |
| **分享能力** | 强（群/朋友圈） | 中（依赖JS-SDK自定义分享） |
| **框架选择** | 原生/uni-app/Taro等 | Vue/React/原生HTML，完全自由 |
| **开发成本** | 中高 | 低（Web标准） |

## 二、H5上线必须做的事

### 第一步：基础资质（和做网站一样）

| # | 要准备什么 | 去哪办 | 耗时 | 费用 |
|---|-----------|--------|------|------|
| 1 | **营业执照**（企业主体） | 当地市场监管局 | 1-2周 | 0-500元 |
| 2 | **对公银行账户** | 银行 | 3-5天 | 免费 |
| 3 | **域名购买** | 阿里云/腾讯云 | 10分钟 | 几十元/年 |
| 4 | **SSL证书** | 云服务商免费申请 | 10分钟 | 免费 |
| 5 | **ICP备案**（域名备案） | 通过云服务商提交通管局 | 约20个工作日 | 免费 |
| 6 | **公安联网备案** | 全国互联网安全管理服务平台 | ICP备案后30日内 | 免费 |

> ⚠️ ICP备案完成之前，域名无法在国内服务器正常运行。这是**真正卡你上线**的东西，建议第一个办。

### 第二步：服务号配置（微信特有）

| # | 要准备什么 | 去哪办 | 耗时 | 费用 |
|---|-----------|--------|------|------|
| 7 | **注册服务号**（企业类型） | 微信公众平台 | 1-3天 | 0 |
| 8 | **服务号微信认证** | 公众平台→设置→微信认证 | 3-5天 | 300元/年 |
| 9 | **配置网页授权回调域名** | 公众平台→设置与开发→功能设置→网页授权域名 | 10分钟 | 0 |
| 10 | **配置JS接口安全域名** | 公众平台→设置与开发→功能设置→JS接口安全域名 | 10分钟 | 0 |
| 11 | **绑定开发者微信号** | 微信开发者平台→账号管理→成员管理→服务号 | 2分钟 | 0 |

#### 关键说明

**网页授权回调域名**：用户通过微信打开你的H5时，需要走OAuth授权获取openid。你填写的就是这个OAuth回调的域名根。格式：`your-domain.com`（不加 http://，纯域名）。

**JS接口安全域名**：如果你想用微信JS-SDK（自定义分享、扫码、获取位置等），需要在这里绑定域名。最多5个。

### 第三步：支付接入

| # | 要准备什么 | 去哪办 | 耗时 | 费用 |
|---|-----------|--------|------|------|
| 12 | **申请微信支付商户号** | pay.weixin.qq.com | 1-7个工作日 | 0 |
| 13 | **开通JSAPI支付权限** | 商户平台→产品中心→JSAPI支付 | 1-7个工作日 | 0 |
| 14 | **商户号绑定服务号AppID** | 支付申请时一步完成 | 同步 | 0 |
| 15 | **后端实现支付接口** | 你的服务器 | 按开发进度 | 0.6%手续费 |

#### 支付开通的核心要求

- 必须用**已认证的服务号**AppID（政府、事业单位、媒体类公众号也可）
- 商户主体与公众号主体必须一致
- 申请时需提供公众号页面截图（展示商品/服务的截图，未上线可用设计稿）
- 客服电话必须真实有效

> 🔑 **最大的利好：没有Apple 12%的抽成。** JSAPI支付在iOS和Android上走同一个费率通道（约0.6%），100元会员费实收约99.4元。

### 第四步：技术开发侧

| # | 要做的事 | 说明 |
|---|---------|------|
| 16 | **搭建后端服务** | Node.js / Python / Go 等，或者直接用微信云开发 |
| 17 | **开发H5前端页面** | Vue/React/原生，完全自由 |
| 18 | **接入网页授权（OAuth）** | 用户从服务号菜单进入→跳转微信OAuth页面→点击同意→回调获取openid |
| 19 | **接入JS-SDK（可选）** | 如需自定义分享内容、扫码等功能；后端需实现签名算法 |
| 20 | **接入JSAPI支付** | 后端下单 → 前端 `wx.chooseWXPay` 拉起收银台 → 回调处理 |
| 21 | **AI生成内容标识** | 在AI回复下方标注"**本内容由AI生成，仅供参考**" |
| 22 | **内容安全过滤** | 接入微信内容安全API或自建敏感词库 |
| 23 | **用户协议 + 隐私政策** | H5页面底部显眼位置挂链接 |

## 三、H5可以不做的（和小程序对比）

| 可以不做的事 | 原因 |
|-------------|------|
| 深度合成类目申请 | 服务号没有"类目"概念，不存在这个东西 |
| 上传算法备案凭证给微信 | 微信不审核你的H5代码，不查这个 |
| 小程序代码审核 | H5不经过微信代码审查 |
| Apple开发者账号 | 不走Apple订阅通道 |

## 四、法律层面你仍然需要注意的

虽然微信不查H5，但以下法律义务依然存在：

### 4.1 算法备案（深度合成服务备案）

**法律依据**：《互联网信息服务深度合成管理规定》《生成式人工智能服务管理暂行办法》

**法律要求**：面向中国境内公众提供AI生成内容服务，需要完成算法备案。

**现实情况**：微信不查H5，网信办也极少主动扫单个H5页面。大量AI产品在H5上运行、提供会员付费，并未完成自研算法备案。但这在法律上是**灰色、甚至偏黑的**。

**给你的操作建议**（和上次一样）：
- 你不是自研模型，你调的是通义千问/DeepSeek等三方API
- 作为调用方，仍可能被认定为"深度合成服务提供者"
- 最稳妥的做法：仍然去阿里云百炼下载一份算法备案证书 + 合作协议，**留档备用**
- 如果未来网信办来查，你有文件证明你的AI能力来源是已备案的大厂，比两手空空好得多

### 4.2 ICP经营许可证

和之前分析的一致：有会员收费 → 法律上算"经营性" → 理论上需要。但实际执行中H5网站被主动查的概率极低。建议上线后补办。

### 4.3 等保（网络安全等级保护）

如果你的H5收集用户手机号等信息，且用户量上来后，建议做等保二级。刚上线时可以暂缓。

## 五、模拟一个典型的用户流程

以你的AI解题场景为例：

```
1. 用户关注你的服务号
2. 用户点击底部菜单「AI解题」
3. 微信内置浏览器打开你的H5：
   https://your-domain.com/ai-solve
4. 第一次使用时，微信弹出OAuth授权页：
   "XXX服务号 申请获取你的昵称和头像"
   用户点击「允许」
5. 回调到你的H5，你拿到 openid
6. 你在后端建立用户账户，绑定 openid
7. 用户提问 → 后端调 DeepSeek API → 返回结果
8. 免费次数用完 → 弹出付费弹窗
9. 用户点击「开通会员」
10. 后端调用微信统一下单API → 前端调起 wx.chooseWXPay
11. 用户输入支付密码 → 支付成功
12. 后端收到支付回调 → 开通会员权益
```

## 六、开发技术清单（JavaScript技术栈示例）

### 6.1 后端需要实现的接口

```
POST   /api/auth/wechat          — 微信OAuth回调，用code换openid，生成JWT
GET    /api/user/info            — 获取用户信息（会员状态、剩余次数）
POST   /api/ai/ask               — AI解题接口（调DeepSeek/通义千问）
POST   /api/pay/create-order      — 创建支付订单
POST   /api/pay/notify            — 微信支付异步回调（签名验证+开通会员）
GET    /api/member/status         — 查询会员状态
POST   /api/wechat/jsapi-sign     — 获取wx.config签名（如使用JS-SDK）
GET    /jsapi-callback             — JS接口安全域名验证文件
```

### 6.2 前端关键代码片段

**OAuth授权跳转：**

```js
// 用户进入页面，没有登录态时，引导授权
const appId = 'wx1234567890abcdef';
const redirectUri = encodeURIComponent('https://your-domain.com/auth/callback');
const scope = 'snsapi_userinfo';  // 弹窗授权，拿openid+用户信息
// const scope = 'snsapi_base';   // 静默授权，只拿openid

const authUrl = `https://open.weixin.qq.com/connect/oauth2/authorize?appid=${appId}&redirect_uri=${redirectUri}&response_type=code&scope=${scope}&state=STATE#wechat_redirect`;

window.location.href = authUrl;
```

**JSAPI支付调起：**

```js
// 后端下单后返回支付参数
async function pay(orderData) {
  if (typeof WeixinJSBridge === 'undefined') {
    // 微信环境内才有效
    alert('请在微信中打开');
    return;
  }

  WeixinJSBridge.invoke('getBrandWCPayRequest', {
    appId: orderData.appId,
    timeStamp: orderData.timeStamp,
    nonceStr: orderData.nonceStr,
    package: orderData.package,
    signType: orderData.signType,
    paySign: orderData.paySign,
  }, (res) => {
    if (res.err_msg === 'get_brand_wcpay_request:ok') {
      // 支付成功，查询订单确认
      checkOrderStatus(orderData.orderId);
    }
  });
}
```

> 注意：JSAPI支付不需要引入JS-SDK文件，`WeixinJSBridge` 是微信内置浏览器自带的。但如果你要用分享、扫码等功能，才需要走 `wx.config`。

```

第5周：    申请微信支付商户号 + 开通JSAPI支付
第6周：    联调支付 + OAuth登录 + AI接口
第7-8周：  测试 + 完善 + 正式上线
第8周后：  上线后30日内完成公安联网备案

（全程无需等待微信类目审核、无需等待算法备案审批）
```

## 八、总结：H5路径的真正优势

| 优势 | 具体体现 |
|------|---------|
| **审核免扰** | 微信不审你的代码、不查你的类目、不管你的AI功能 |
| **没有Apple税** | JSAPI支付统一0.6%费率，100元会员费实收99.4元 |
| **开发自由** | HTML/CSS/JS标准技术栈，什么框架都能用 |
| **快速上线** | 不被类目审核/算法备案卡时间，域名备案完就能上 |
| **降低成本** | 无需小程序认证费外的额外资质费用 |

| 劣势 | 具体体现 |
|------|---------|
| **用户体验** | H5加载速度不如小程序原生，每次打开需加载网页 |
| **留存弱** | 没有桌面快捷方式、下拉入口、模板消息等留存能力 |
| **分享弱** | 分享到群/朋友圈体验不如小程序 |
| **合规风险** | 虽然微信不查，但法律上的算法备案义务依然存在 |

---

> **最终建议：先用服务号H5快速验证产品、跑通付费模型（省下Apple的12%）。产品数据验证通过后，再同步上架小程序版本，用小程序做增长和留存。两个渠道可以共享同一套后端和会员体系。**
